## 1. 项目背景

电商首饰产品需要大量"模特上身图"（即模特佩戴首饰的产品图）。  
传统流程需要拍摄、修图，成本高、周期长。  
本工具通过 **Gemini Image Editing API**，将现有模特图与首饰原图自动合成产品图，大幅降低人力成本，提升出图效率。

## 2. 目标用户

- 电商运营人员
- 产品图设计师

## 3. 功能目标

### 3.1 MVP 目标（第一阶段）

| 功能 | 说明 |
|------|------|
| 读取云盘图片 | 从阿里云盘挂载 Windows 盘符路径读取模特图和产品原图 |
| 固定模特模式 | 指定单张模特图，批量与所有首饰图合成 |
| 项链合成 | 将首饰图中的项链合成到模特颈部，输出产品图 |
| 批量处理 | 支持单次处理数百张首饰图，带并发队列控制 |
| 本地输出 | 合成图保存到本地指定输出目录，文件名与原图对应 |

### 3.2 后续迭代目标（第二阶段）

| 功能 | 说明 |
|------|------|
| 多首饰类型 | 支持耳环、戒指、手链等不同类型，自动选择对应 prompt |
| 首饰类型识别 | 自动识别首饰类型（可通过文件夹或 Gemini Vision 识别） |
| 多模特支持 | 支持多个模特图与首饰图的配对策略 |
| Web UI | 可视化任务管理界面，支持拖拽、预览、下载 |
| 结果审核 | 生成结果缩略图预览页，方便人工快速审核 |

## 4. 文件路径规范

### 4.1 输入路径

| 类型 | 路径示例 |
|------|---------|
| 模特图 | `G:\阿里云盘\电商\模特图\female\002.jpeg` |
| 首饰原图 | `G:\阿里云盘\电商\产品_待生成\20260228\xxx.jpg` |

- 模特图目录下可有多张图，MVP 阶段手动在配置文件中指定使用哪张
- 首饰图按日期文件夹归档，每次任务指定一个日期文件夹

### 4.2 输出路径

| 类型 | 路径示例 |
|------|---------|
| 生成产品图 | `G:\阿里云盘\电商\产品_已生成\20260228\xxx_result.jpg` |

- 输出目录自动创建（按任务日期命名子文件夹）
- 首饰图目录内直接存放 `.jpg` / `.jpeg` 文件（无子文件夹）

## 5. 技术架构

### 5.1 技术栈

| 层级 | 技术 |
|------|------|
| 语言 | TypeScript (Node.js 20+) |
| AI API | Google Gemini API (`@google/genai`) |
| 配置管理 | `dotenv`（API Key 等） |
| 并发控制 | `p-limit`（限制并发 API 请求数） |
| 运行方式 | `tsx`（直接运行 TS，无需编译） |

> 首饰产品图需保持原始高清品质，图片**不做任何压缩处理**，直接以原始 bytes 上传至 API。

### 5.2 模块划分

```
src/
├── config.ts            # 路径配置、API Key、并发数、模型选择
├── scanner.ts           # 扫描首饰图目录，返回图片文件列表
├── geminiClient.ts      # 封装 Gemini image editing API 调用
├── promptTemplates.ts   # 不同首饰类型的 prompt 模板
├── pipeline.ts          # 串联：扫描 → 合成 → 保存，带进度输出
└── main.ts              # CLI 入口，读取 config 执行 pipeline
```

### 5.3 调用流程

```
main.ts
  └─ 读取 config（模特图路径、首饰图目录、输出目录）
       └─ scanner.ts 扫描首饰图目录 → 返回图片列表
            └─ pipeline.ts（p-limit 控制并发）
                 ├─ 对每张首饰图：
                 │    ├─ 读取模特图（固定）
                 │    ├─ 读取首饰图
                 │    ├─ 图片预处理（sharp 压缩）
                 │    ├─ geminiClient.ts 调用 API
                 │    └─ 保存输出图片
                 └─ 完成后输出统计（成功/失败数量）
```

## 6. Gemini API 使用规范

### 6.1 模型选择

| 场景 | 模型 |
|------|------|
| 日常批量，性价比优先 | `gemini-3.1-flash-image-preview` |
| 高质量精品图（可选） | `gemini-3-pro-image-preview`（支持 4K） |

### 6.2 调用方式

- 模式：**Image Editing（text-and-image-to-image）**
- 输入：模特图（base64）+ 首饰图（base64）+ prompt 文字
- 输出：合成后产品图（base64 → 保存为 JPEG）

### 6.3 Prompt 模板管理

- 每种首饰类型对应 `prompts/{type}.txt` 文件
- 程序运行时根据 `config.ts` 中的 `jewelryType` 自动读取对应文件
- Prompt 使用英文编写（Gemini API 英文模型效果最佳）
- 项链指令要点：首饰放置位置、保留模特原始态、首饰细节忽实、光影自然融合、高清输出

## 7. 配置文件设计（`config.ts`）

```typescript
export const config = {
  // Gemini API Key（从 .env 读取）
  apiKey: process.env.GEMINI_API_KEY!,
  
  // 固定模特图完整路径
  modelImagePath: "G:\\阿里云盘\\电商\\模特图\\female\\002.jpeg",
  
  // 首饰图目录（按日期）
  jewelryDir: "G:\\阿里云盘\\电商\\产品_待生成\\20260228",
  
  // 输出目录
  outputDir: "./outputs/20260228",
  
  // 并发请求数（可配置，默认 5，避免 API Rate Limit 建议不超过 10）
  concurrency: 5,
  
  // 首饰类型（决定使用哪个 prompt）
  jewelryType: "necklace" as "necklace" | "ring" | "earring",
  
  // 输出根目录（写回阿里云盘）
  outputBaseDir: "G:\\阿里云盘\\电商\\产品_已生成",

  // 模型选择
  model: "gemini-3.1-flash-image-preview",
};
```

## 8. 非功能性要求

| 项目 | 要求 |
|------|------|
| 并发控制 | 默认最多 5 个并发 API 请求（可通过 config 修改） |
| 错误处理 | 单张图片失败不中断全局任务，记录失败列表 |
| 进度输出 | CLI 实时显示进度：已处理 / 总数 / 失败数 |
| 图片质量 | **不压缩**，保持原始高清画质直接上传 |
| 重试机制 | API 失败自动重试 2 次（指数退避） |
| 日志 | 每次任务生成 `run.log` 记录详情（输出目录内） |

## 11. 已知风险 & 约束

| 风险 | 说明 | 缓解措施 |
|------|------|---------|
| Gemini API Rate Limit | 高并发时触发 429 限流 | `p-limit` 限并发 + 指数退避重试 |
| 合成质量不稳定 | API 输出结果有随机性 | 优化 prompt；可选择 Pro 模型 |
| 图片尺寸过大 | API 有 payload 限制 | `sharp` 预处理压缩 |
| 阿里云盘掉线 | 网络盘偶发断连 | 文件读取失败时跳过并记录 |
| 版权合规 | 模特/首饰图须有使用授权 | 由使用方保证图片授权合规 |
